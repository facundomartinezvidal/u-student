# u-student

Claude Code plugin for university and course students. Study, summarize course material, create presentations, write academic documents, and prepare for exams — always grounded in your own course content.

Always responds in Spanish and prioritizes your own course material over generic knowledge.

## Installation

Requires [Claude Code](https://claude.ai/code).

```bash
claude plugin marketplace add facundomartinezvidal/claude-plugins
claude plugin install u-student@fmartinezvidal-plugins --scope user
```

## Getting started

Open Claude Code in your subject folder and run:

```
/u-student:init
```

This creates the folder structure, the course schedule template, and a `CLAUDE.md` file that loads automatically every session so Claude knows which subject you're working on.

## Workspace structure

```
mi-materia/
├── CLAUDE.md                    # Auto-loaded subject context (generated by init)
├── cronograma.md                # Class schedule — maps class numbers to topics and files
├── programa.md                  # Official course syllabus
└── content/
    ├── primer-parcial/
    │   ├── 01-introduccion.pdf
    │   └── 02-tema-dos.md
    └── segundo-parcial/
        └── 03-tema-tres.pdf
```

Name files as `NN-topic-name.ext` so the plugin automatically matches them to their class in the cronograma.

## Commands

### `/u-student:help`
Shows a full guide of all commands, agents, and recommended workflow — in Spanish.

---

### `/u-student:init`
Initializes the subject workspace. Run this first in your subject folder.

- Collects subject metadata (name, professor, career, year, number of parciales)
- Creates `content/primer-parcial/` and `content/segundo-parcial/` (or more, depending on your setup)
- Loads your existing cronograma and programa if you have them
- Generates `CLAUDE.md` with full subject context and folder tree
- Detects if Git is installed — skips Git setup if not found

---

### `/u-student:organize`
Scans your folder for loose files (PDFs, slides, notes) and moves them into the standard structure.

- Cross-references filenames against the cronograma to assign each file to the right class
- Proposes a full rename/move plan before executing — nothing moves without your approval
- Updates the cronograma's file column and regenerates the CLAUDE.md folder tree

---

### `/u-student:summarize [topic]`
Summarizes course material using your own files as the primary source.

- Choose scope: full course, specific parcial, or specific classes
- Choose format: standard, bullet points, extended, per-class, or per-unit
- Choose depth: short / medium / long
- Optionally generates self-evaluation questions (open or multiple choice)
- Flags any class from the cronograma that has no matching material file:
  `⚠️ Clase 04 — "Transformadas": no se encontró material.`

---

### `/u-student:study [concept]`
Guided step-by-step session to understand a concept or work through a problem.

- Detects the type of confusion: conceptual, procedural, relational, exercise-based, or overwhelmed
- Never gives the answer directly — guides you to reach it yourself
- After confirming understanding, offers to continue with quiz, flashcards, explain, or mindmap

---

### `/u-student:explain [concept]`
Feynman technique — you explain the concept in your own words, Claude evaluates it.

- Identifies exactly what you understood correctly and what's still unclear
- Points to the specific gap, not just "you're wrong"
- Lets you try again after feedback

---

### `/u-student:compare [concept A] [concept B]`
Generates a structured comparison table between two or more concepts.

- Highlights key differences and similarities
- Adds a synthesis section
- Flags common mistakes students make when confusing the two

---

### `/u-student:mindmap [topic]`
Generates a Mermaid mind map of course concepts, organized by unit or parcial.

- Paste the output in any Mermaid-compatible viewer (Obsidian, Notion, VS Code extension, etc.)
- Can scope to full course, one parcial, or a specific unit

---

### `/u-student:quiz [--parcial 1|2]`
Interactive quiz — questions are shown one at a time without visible answers.

- You respond, then get immediate feedback: ✅ / ⚠️ / ❌ with explanation
- Tracks score across the session
- Based entirely on your course material and cronograma

---

### `/u-student:flashcards [topic]`
Classic flashcard session with score tracking.

- Shows question → you respond → answer is revealed
- Tracks what you got right and wrong
- At the end, recommends which topics to review

---

### `/u-student:exam [--parcial 1|2]`
Full mock partial exam — simulates the real experience.

- Shows all questions upfront (you can't go back mid-exam)
- After submitting all answers, grades each one with detailed correction
- Gives an estimated grade (1–10 scale) with breakdown by topic
- Can analyze past exams to match your professor's question style (use `exam-analyst` agent first)

---

### `/u-student:presentation [topic]`
Creates a structured HTML presentation ready to export as PDF.

- Asks for exposition time → calculates slide count automatically (1.5 min/slide)
- Accepts professor guidelines or assignment brief (consigna)
- First generates a slide brief (table: # | Title | Key point) — waits for your approval
- Then generates full HTML with dark theme, speaker notes, and print CSS for PDF export
- Next step: `/u-student:export` to PDF, or `/u-student:defense` to prepare the oral

---

### `/u-student:defense [--plan | --simulate]`
Prepares you for the oral defense of a presentation.

- `--plan`: written defense guide with what to say per slide, transitions, and likely professor questions
- `--simulate`: interactive slide-by-slide simulation — you present in writing, get feedback per slide, final overall summary

---

### `/u-student:write [type]`
Writes TPs, reports, and essays **section by section** — never all at once.

- Types: `tp`, `informe`, `ensayo`, `reporte`
- Reads the assignment guidelines (consigna) first — asks if not provided
- Enters plan mode to define document structure, format (md/docx), length, and citation style
- Writes one section → waits for your approval → moves to the next
- Marks gaps you need to fill with `[COMPLETAR: descripción]`
- Offers `.docx` export via pandoc

---

### `/u-student:export [file]`
Exports a document or presentation to PDF.

- Supports `.html`, `.md`, `.docx`
- Tries tools in order: Chrome headless → pandoc → wkhtmltopdf → LibreOffice
- If no tool is available, gives step-by-step manual instructions (browser print dialog)

```bash
/u-student:export presentacion.html
/u-student:export informe-final.md
/u-student:export tp-entrega.docx
```

## Agents

Auto-triggered based on what you write — no need to call them explicitly.

| Agent | Triggers on | What it does |
|-------|------------|--------------|
| `tutor` | "no entiendo", "explicame", "qué es X" | Detects confusion type, finds relevant material, guides understanding step by step |
| `study-planner` | "tengo el parcial en X días", "armame un plan" | Asks your availability, builds a day-by-day study plan, saves it to `plan-estudio-parcial.md` |
| `tp-reviewer` | "revisá mi TP", "corregí el trabajo" | Reviews section by section, checks against assignment guidelines, estimates grade (1–10) |
| `writing-coach` | "cómo empiezo", "suena bien esto?", "cómo redacto" | Guides you to write yourself — never rewrites unprompted. Offers 2 versions only when explicitly asked |
| `research-assistant` | "dónde busco fuentes", "esta fuente sirve?" | Checks programa.md bibliography first, rates source credibility, provides APA and Chicago citations |
| `exam-analyst` | "tengo parciales viejos", "qué toman siempre" | Analyzes past exams for topic frequency and professor style, generates practice questions |
| `progress-tracker` | "cuánto me falta", "cómo voy" | Cross-references cronograma with content/ files, shows visual progress bar, saves state to `progreso.md` |

## Recommended workflow

```bash
# 1. Initialize your subject
/u-student:init

# 2. Add materials to content/primer-parcial/ and content/segundo-parcial/
#    Naming convention: 01-topic-name.pdf

# 3. Organize any loose files
/u-student:organize

# 4. Study
/u-student:summarize
/u-student:quiz --parcial 1
/u-student:exam --parcial 1

# 5. Create deliverables
/u-student:presentation
/u-student:write tp
/u-student:export presentacion.html
```

- [Claude Code](https://claude.ai/code)
